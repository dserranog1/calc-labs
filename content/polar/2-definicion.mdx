## ¬øQu√© representan $r$ y $\theta$?

En el sistema de **coordenadas polares**, cada punto del plano se describe por una pareja ordenada:

```math
(r, \theta)
```

donde:

- $r$ es la **distancia desde el origen** (el punto $(0, 0)$) hasta el punto.
- $\theta$ es el **√°ngulo** entre el eje $x$ positivo y la l√≠nea que conecta el origen con el punto, medido en **radianes** en sentido **antihorario**.

---

### üß≠ ¬øC√≥mo interpretamos $(r, \theta)$?

- Si $r > 0$, nos movemos $r$ unidades desde el origen en la direcci√≥n del √°ngulo $\theta$.
- Si $r = 0$, estamos en el origen sin importar el valor de $\theta$.
- Si $r < 0$, nos movemos $|r|$ unidades en la direcci√≥n **opuesta** a la indicada por $\theta$.

El √°ngulo $\theta$ puede tomar **valores mayores a $2\pi$** o ser **negativo**, lo que representa m√∫ltiples vueltas alrededor del origen.

---

### ‚ö†Ô∏è Representaciones m√∫ltiples del mismo punto

Una misma posici√≥n en el plano puede tener distintas representaciones en coordenadas polares. Por ejemplo:

```math
(1, \pi) = (-1, 0)
```

Ambos representan el mismo punto: una unidad en direcci√≥n opuesta.

Esto significa que las coordenadas polares **no son √∫nicas**. A veces conviene normalizar $\theta$ en el intervalo $[0, 2\pi)$ o limitar $r \geq 0$ seg√∫n el contexto.

---

### üìä Visualizaci√≥n: ¬øC√≥mo se ve un punto polar?

El siguiente c√≥digo grafica varios puntos dados en coordenadas polares, para que puedan visualizar c√≥mo se interpretan $r$ y $\theta$ en el plano.

```python
import numpy as np
import matplotlib.pyplot as plt

# Lista de puntos en coordenadas polares: (r, Œ∏)
puntos_polares = [
    (1, np.pi/4),
    (2, np.pi),
    (1, 5*np.pi/2),  # equivale a (1, œÄ/2)
    (-1, 0),         # equivale a (1, œÄ)
    (0, 2*np.pi/3)   # origen
]

fig, ax = plt.subplots(figsize=(6,6))
ax.set_aspect('equal')
ax.set_xlim(-2.5, 2.5)
ax.set_ylim(-2.5, 2.5)
ax.set_title("Interpretaci√≥n geom√©trica de $(r, \\theta)$")
ax.grid(True)

# Dibujar ejes
ax.axhline(0, color='gray', linewidth=1)
ax.axvline(0, color='gray', linewidth=1)

# Graficar puntos
for r, theta in puntos_polares:
    x = r * np.cos(theta)
    y = r * np.sin(theta)
    ax.plot(x, y, 'o')
    ax.annotate(f"({r:.1f}, {theta:.2f})", (x, y), textcoords="offset points", xytext=(5,5))

plt.show()
```

---

### üß† Observaciones importantes

- El punto $(1, \pi)$ est√° en el eje negativo $x$, igual que $(-1, 0)$.
- El punto $(1, 5\pi/2)$ est√° en la misma direcci√≥n que $(1, \pi/2)$.
- El punto $(0, \theta)$ siempre es el **origen**, sin importar $\theta$.

## üîÑ Conversi√≥n entre coordenadas cartesianas y polares

En muchos problemas necesitaremos convertir un punto de **coordenadas cartesianas** $(x, y)$ a **coordenadas polares** $(r, \theta)$, y viceversa.

---

### üìê De polares a cartesianas

Dado un punto en coordenadas polares $(r, \theta)$, sus coordenadas cartesianas se obtienen con:

```math
x = r \cos(\theta), \quad y = r \sin(\theta)
```

Estas f√≥rmulas se derivan directamente de trigonometr√≠a, considerando $r$ como la hipotenusa de un tri√°ngulo rect√°ngulo.

---

### üìê De cartesianas a polares

Dado un punto $(x, y)$, sus coordenadas polares se calculan as√≠:

```math
r = \sqrt{x^2 + y^2}, \quad \theta = \tan^{-1}\left(\frac{y}{x}\right)
```

‚ö†Ô∏è Ojo! Esta f√≥rmula para $\theta$ puede ser ambigua, ya que la tangente no distingue en qu√© cuadrante est√° el punto. Por eso en programaci√≥n se recomienda usar:

- `atan2(y, x)` en Python (de `numpy` o `math`), que devuelve el √°ngulo correcto considerando el signo de ambos argumentos.

---

### üßÆ Ejemplo de conversi√≥n

Supongamos que tenemos el punto cartesiano $(x, y) = (-1, \sqrt{3})$.  
Entonces:

```math
r = \sqrt{(-1)^2 + (\sqrt{3})^2} = \sqrt{1 + 3} = 2
\\
\theta = \text{atan2}(\sqrt{3}, -1) = \frac{2\pi}{3}
```

Por tanto, el punto en coordenadas polares es $(2, 2\pi/3)$.

---

### üß™ Visualizaci√≥n interactiva


```python
import numpy as np
import matplotlib.pyplot as plt

# Lista de puntos cartesianos
puntos_cartesianos = [
    (1, 1),
    (0, -2),
    (-1, np.sqrt(3)),
    (-2, -2)
]

fig, ax = plt.subplots(figsize=(6,6))
ax.set_aspect('equal')
ax.set_xlim(-3, 3)
ax.set_ylim(-3, 3)
ax.set_title("Conversi√≥n cartesiano ‚Üí polar")
ax.grid(True)

# Dibujar ejes
ax.axhline(0, color='gray', linewidth=1)
ax.axvline(0, color='gray', linewidth=1)

# Dibujar puntos y mostrar conversi√≥n
for x, y in puntos_cartesianos:
    r = np.sqrt(x**2 + y**2)
    theta = np.arctan2(y, x)
    x_label = f"x={x}, y={y}"
    rtheta_label = f"r={r:.2f}, Œ∏={theta:.2f} rad"
    ax.plot(x, y, 'o')
    ax.annotate(rtheta_label, (x, y), textcoords="offset points", xytext=(5, 5))
    print(f"{x_label} ‚Üí {rtheta_label}")

plt.show()
```

---

### üí¨ Observaciones importantes

- `atan2` devuelve $\theta$ en radianes, entre $-\pi$ y $\pi$. Esto se puede ajustar eso al intervalo $[0, 2\pi)$.
- Algunos puntos (como el origen) tienen coordenadas polares **mal definidas**: $r = 0$, pero $\theta$ puede ser cualquier valor.


---

### ‚ùó ¬øPor qu√© la f√≥rmula de $\theta$ puede fallar?

Cuando convertimos un punto cartesiano $(x, y)$ a coordenadas polares, usamos:

```math
\theta = \tan^{-1}\left(\frac{y}{x}\right)
```

Pero esta f√≥rmula **es ambigua**, porque la tangente **no distingue en qu√© cuadrante est√° el punto**.  
Por ejemplo:

```math
\tan^{-1}\left(\frac{1}{1}\right) = \frac{\pi}{4}
\\
\tan^{-1}\left(\frac{-1}{-1}\right) = \frac{\pi}{4}
```

Ambos dan el mismo resultado, pero:

- $(1, 1)$ est√° en el **primer cuadrante**
- $(-1, -1)$ est√° en el **tercer cuadrante**

Esto se debe a que $\tan^{-1}(y/x)$ s√≥lo mira el cociente, **no los signos por separado**.

---

### ‚úÖ Soluci√≥n: usar `atan2(y, x)`

La funci√≥n `atan2(y, x)` (disponible en Python, C, Java, etc.) **resuelve este problema**:

- Usa **el signo de $x$ y $y$** para determinar el cuadrante correcto.
- Devuelve el valor de $\theta$ en radianes, en el intervalo $(-\pi, \pi]$.

---

### üîç Comparaci√≥n visual

El siguiente c√≥digo muestra dos puntos que tienen el mismo $\frac{y}{x}$ pero est√°n en cuadrantes distintos.  
Se puede ver que $\tan^{-1}(y/x)$ no los diferencia, pero `atan2(y, x)` s√≠.

```python
import numpy as np
import matplotlib.pyplot as plt

# Dos puntos con mismo y/x pero distinto cuadrante
p1 = (1, 1)     # Cuadrante I
p2 = (-1, -1)   # Cuadrante III

# Calcular √°ngulos
theta1 = np.arctan2(p1[1], p1[0])
theta2 = np.arctan2(p2[1], p2[0])

# Mostrar valores
print(f"Punto (1, 1): Œ∏ = {theta1:.2f} rad = {np.degrees(theta1):.1f}¬∞")
print(f"Punto (-1, -1): Œ∏ = {theta2:.2f} rad = {np.degrees(theta2):.1f}¬∞")

# Graficar
fig, ax = plt.subplots(figsize=(6,6))
ax.set_aspect('equal')
ax.set_xlim(-2, 2)
ax.set_ylim(-2, 2)
ax.grid(True)
ax.axhline(0, color='gray', linewidth=1)
ax.axvline(0, color='gray', linewidth=1)

for x, y in [p1, p2]:
    ax.plot(x, y, 'o')
    ax.arrow(0, 0, x, y, head_width=0.1, head_length=0.1, color='orange', length_includes_head=True)

ax.annotate("Œ∏ = œÄ/4", p1, textcoords="offset points", xytext=(5,5))
ax.annotate("Œ∏ = 5œÄ/4", p2, textcoords="offset points", xytext=(5,5))

ax.set_title("Mismo y/x ‚Üí distinta direcci√≥n")
plt.show()
```

---

Este es un ejemplo cl√°sico de c√≥mo **la geometr√≠a importa m√°s que la f√≥rmula algebraica**.  
En resumen:

- Usar `atan2(y, x)` siempre que quieras obtener $\theta$ correctamente.
- No conf√≠ar ciegamente en $\tan^{-1}(y/x)$
